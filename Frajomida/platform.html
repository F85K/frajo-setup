How the platform works

A simple flow: create a tenant, run security checks, deploy a container, and keep each client isolated with their own database and credentials.

Deployment steps

  1. Create tenant: admin provisions a dedicated database and API key.
  2. Push code: client connects their Git repo (or uploads an image).
  3. Scan: automated checks (e.g., Trivy) run before deployment.
  4. Deploy: container starts with the tenant’s DB credentials injected.
  5. Monitor: clients see their own status, logs, and scan reports—nothing else.

Key guarantees

  • Isolation: per-tenant database and API key, no cross-tenant access.
  • Security: scans gate deployments; approvals stay with admins.
  • Simplicity: one API to provision, scan, and deploy.
  • Observability: reports and logs per tenant.
